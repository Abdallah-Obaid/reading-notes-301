# Database normalization:-
----------------------------------
 * **Database normalization** is a process used to organize a database into tables and columns.

 * The main idea of **Database normalization** is that a table should be about a specific topic and only supporting topics included.
 * We should limiting a table to one purpose by reduce the number of duplicate data contained within our database.
 * To achieve these objectives, weâ€™ll use some established rules. As you apply these rules, new tables are formed.

 ### Reasons for Database Normalization:-
 ----------------------------------
 * minimize duplicate data.
 * minimize or avoid data modification issues
 * simplify queries. 

 ### Data Duplication and Modification Anomalies:-
  ----------------------------------
 ##### Duplicated information presents two problems:-
 * It increases storage and decrease performance.
 * It becomes more difficult to maintain data changes.
